# 📊 Data Suite Visualization Guide

This guide explains every plot we have implemented in the suite, which models they apply to, and the technical "why" behind them.

---

## 1. Classification Suite
*Applied to: Random Forest, Decision Tree, Logistic Regression, KNN, SVM*

### 🧱 Confusion Matrix (Heatmap)
*   **Why**: It is the "truth table" for classification. It shows exactly how many samples were correctly predicted vs. how many were misclassified into other labels. It helps identify if the model is biased toward a specific class.

### 🗺️ PCA Decision Boundaries (Contour Map)
*   **Why**: Machine learning often happens in high dimensions (e.g., 10 features). We use PCA to squish these into 2 dimensions so you can visually see the "territory" of each class. It helps you see if your model is "overfitting" (jagged boundaries) or "underfitting" (too simple).

### 📈 ROC Curve & AUC
*   **Why**: This measures the model's ability to distinguish between classes. A perfect curve stays in the top-left corner. It's the industry standard for measuring classifier quality regardless of class imbalance.

### 📉 Feature Coefficients/Importance (Bar Charts)
*   **Why**: 
    *   **Coefficients (Logistic)** show the *direction* (Does this feature push the result up or down?).
    *   **Importance (Trees)** show the *weight* (How much does the model rely on this feature to make a split?).

## 2. Regression Suite
*Applied to: Linear Regression, RF Regressor, DT Regressor, etc.*

### 🎯 Actual vs. Predicted (Scatter + Trendline)
*   **Why**: This is the ultimate visual test for regression. In a perfect world, every point would sit on the diagonal line. It helps you see if the model is consistently overestimating or underestimating values at certain ranges.

### 📉 Residual Plot
*   **Why**: Errors (residuals) should be random. If you see a shape (like a U-shape or a funnel) in this plot, it means your data has a non-linear pattern that your current model is failing to capture.

## 3. Clustering Suite
*Applied to: K-Means*

### 📐 Elbow Method (Inertia Curve)
*   **Why**: Choosing the number of clusters ($k$) can be arbitrary. The Elbow plot calculates "Inertia" (how tight the clusters are). We look for the "elbow" where adding more clusters no longer significantly improves tightness.

### 🧶 PCA Cluster Map
*   **Why**: Since clustering is unsupervised (we don't know the labels beforehand), this plot shows us how the model has naturally grouped the data. It's the best way to identify "segments" in your customer or passenger data.

### 🥧 Cluster Distribution (Pie Chart)
*   **Why**: Helps you quickly see the proportions. If you have 3 clusters but one contains 99% of the data, the clustering might be failing to find meaningful differences.

## 4. Universal Diagnostics
### 🌲 Tree Structural Maps
*   **Why**: (For RF/DT) It lets you "peek inside the brain" of the tree to see the exact numerical logic (e.g., *if Age < 6.5 and Sex is Female, then Survived*).

### 💎 Model Confidence (Prediction Phase)
*   **Why**: Real-world decisions need probability. Telling a doctor "This patient might be sick with 60% confidence" is much more useful than a simple "Yes/No."
